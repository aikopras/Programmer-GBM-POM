Understanding the different kinds of feedback decoder addresses

The decoder's main address

The main address of a feedback decoder is the RS-bus address. This address can be set by pressing the push button on the decoder's PCB. The feedback decoder will then use the first accessory address that it receives on its input. Note that this address should be in the range between 1 and 128.

The main address of a feedback decoder is stored in CV10. This CV (and thus the decoder's address) can be changed via the "CV Tab" of this program.

If the feedback decoder has not yet been initialized, the value stored in CV10 will be 255. This value represents an invalid RS-Bus address, thus a decoder with this CV value will not be able to send any feedback messages over the RS-bus.

If the feedback decoder has not yet been initialized, its main address can also (in addition to pressing the push button on the PCB) be set by using the "Initialize Tab" of this program.

The decoder's loco address

Each feedback decoder listens to locomotive addresses. This address equals the RS-bus address + 6000.

If a feedback decoder has not yet been initialized (the address in CV10 is 255) it will listen to PoM messages at address 6000. Address 6000 is used within the "Initialize Tab" of this program uses to initialize the decoder (set the RS-Bus address in CV10).

In case the feedback decoder is controlling via an extension board four relays, the position of these relays can be changed by using the F1..F4 functions.

The decoder's accessory address

Feedback decoders may also listen to accessory (thus switch) addresses. In this way the relays that can be connected via an extension board can be controlled via "normal" accessory decoder addresses (instead of loco addresses and F1...F4).

The accessory decoder address will be stored in CV1 and CV9 (just like normal accessory decoders).

Note however that for "my" accessory decoder" CV1 should be in the range of 1..64 (Not 0..63!) and CV9 0..3.

In case an accessory decoder address has not yet been initialized, CV9 holds as value 0x80. Initializing the accessory decoder address can be done via the "Relays Tab" of this program.

The decoder only listens to an accessory decoder address if the Decoder Mode is "Relays". This mode can be set via the "Control Tab" of this program.

Acceptable decoder addresses

  • Main address: 1..128 (255 if not yet initialized)
  • Loco address: 6001..6128 (6000 if not yet initialized)
  • Accessory address: 0.255 (>2048 if not yet initialized)
